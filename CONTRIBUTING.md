# Contributing to Target Scheduler

Thank you for considering a contribution to the Target Scheduler NINA plugin. Please keep in mind that this plugin is maintained by one person in their spare time and the ability to concentrate on it will vary over time - so be patient if your request isn't addressed immediately.

If you find an issue with the TS documentation, please open an issue on that repository: https://github.com/tcpalmer/nina-scheduler.

## Issues
Issues can be created and tracked at https://github.com/tcpalmer/nina.plugin.targetscheduler/issues. When submitting a bug report, please include/attach the following:
* The applicable NINA and TS versions
* the relevant NINA and TS log files, with the TS log file at debug level (full logs, not snippets)
* steps to duplicate
* if necessary, your TS database file

## Code Contributions
Contributions are welcome but please follow these rules:
* Open an Issue first: do not submit a Pull Request without discussing it in an issue first. Uncoordinated PRs will be closed.
* All code must be contributed under the terms of the MPL v2 license. NINA and Target Scheduler are free, open-source software and will remain so.
* No AI-generated bulk code: I do not accept large, AI-generated PRs. Code must be human-readable, minimalistic, and match the existing project style.
* Keep it small: PRs should be strictly focused on a single issue.
* Please adhere to the code style and guidelines of the plugin.
* Please add new unit tests as appropriate.
* All tests must pass before submitting a PR.

Bug fixes and thoroughly discussed features are always welcome!
